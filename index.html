The Big Picture — Why Wireless?
📖 The Fundamental Problem

Wireless communication must solve one deceptively hard problem: allow millions of users to share the same physical medium (electromagnetic spectrum) simultaneously, while moving, across a channel that randomly attenuates, delays, and distorts every transmitted signal. Everything in this course is a consequence of this fundamental challenge.

System Architecture — From Radio to Internet
📱 UE → BS Link (Air Interface)
  • UE transmits on the Uplink (UL) to Base Station (BS)
  • BS transmits on the Downlink (DL) to UE
  • Connection via radio signal at carrier frequency f_c
  • BS coverage defines a cell — radius from 300 m (microcell) to 30 km (macrocell)
🏗️ Network Infrastructure
  • BS (Base Station): radio head + antenna — the cell center
  • MSC (Mobile Switching Center): routes calls/data between cells and to PSTN/internet
  • HLR (Home Location Register): permanent subscriber database
  • VLR (Visitor Location Register): tracks UE's current cell location
🔁 Key System Properties
  • Bidirectional: simultaneous TX & RX (FDD) or alternating (TDD)
  • Location independence: network finds the UE — caller needs no location info
  • Handover: seamless transfer between BSs while in active call
  • Privacy: encryption ensures data intended only for recipient
The Cellular Principle — The Core Invention
🔢 Why Cells? The Spectrum Problem
  • Spectrum is finite and shared by all users
  • Divide geographic area into smaller cells
  • Each cell uses only a subset of frequencies
  • Distant cells reuse the same frequencies — beyond interference range
  • Frequency reuse factor K: K=1 (all cells same freq, 5G), K=3,7,… (older cellular)
  • Smaller cells → more cells per km² → higher system capacity
📶 Multiple Access Schemes
  • FDMA: divide spectrum into channels — one user per channel (1G)
  • TDMA: divide time into slots — share each frequency (2G GSM)
  • CDMA: all users share same time & frequency, separated by orthogonal codes (3G)
  • OFDMA: assign OFDM subcarriers per user dynamically (4G LTE, 5G NR)
  • FDD: separate frequencies for UL & DL — continuous transmission
  • TDD: same frequency, alternate UL/DL in time — flexible ratio
Infrastructure Variants — Beyond Terrestrial BSs
🛸 GEO Satellite
  • Altitude: ~36,000 km
  • Large cell diameter (continental)
  • High latency (~250 ms)
  • High power needed
  • Coverage: remote areas
🛰️ LEO Satellite
  • Altitude: 500–2000 km
  • Cell: tens of km diameter
  • Lower latency (<20 ms)
  • StarLink: >5000 satellites
  • Future 6G integration
🚁 UAV / Drone BS
  • Acts as mobile BS in air
  • Adjustable coverage via altitude
  • Events, disaster recovery
  • Challenge: battery, payload
🎙️ Trunking (TETRA)
  • Closed user groups (police, fire)
  • Group calls + call priorities
  • Emergency call preemption
  • Ultra-reliable, push-to-talk
💡 Expert Summary of Module 1

Wireless systems evolved from 6-channel analog systems in 1946 to 5G handling billions of IoT devices simultaneously. The driving innovation is the cellular principle (frequency reuse in smaller cells), which trades infrastructure cost for unlimited capacity scaling. The fundamental architecture (UE → BS → MSC → internet) has remained constant across all generations — only the radio interface technology (modulation, coding, MIMO, OFDM) has evolved. Everything else in this course explains why and how the radio interface works.

The Wireless Channel — Four Core Impairments
🌊 1 — Multipath Propagation (Physical cause of fading)
  • Signal reaches RX via multiple paths: reflections, diffractions, scattering
  • Each MPC has: unique amplitude, delay, phase, angle of arrival/departure
  • Constructive: paths add coherently → signal boost (up to ×2 in amplitude)
  • Destructive: paths cancel → deep fade (theoretically −∞ dB, practically −30 to −40 dB)
  • At 2 GHz, moving just 7.5 cm (= λ/2) can flip constructive to destructive
  • This rapid variation is called small-scale (fast) fading
🏢 2 — Shadowing (Slow amplitude variation)
  • Buildings, hills, walls block or attenuate signal — large-scale variation
  • Changes slowly as UE moves: correlation over 10–100 m (vs λ/2 for fast fading)
  • Modeled as log-normal: S_dB ~ N(0, σ_S²)
  • Typical σ_S: 4–10 dB in urban environments
  • Both fast and slow fading coexist — the total received power = path loss + shadowing + fast fading
📡 3 — Inter-Symbol Interference (ISI)
  • In wideband systems: delayed echoes overlap the next symbol in time
  • Occurs when: delay spread τ_rms > symbol period T_s
  • Example: 1 Msymbol/s → T_s = 1 μs; if τ_rms = 250 ns → ISI from 250 ns-delayed echo!
  • ISI creates an irreducible error floor: increasing TX power cannot fix it
  • Solutions: OFDM (cyclic prefix eliminates ISI), equalizers, RAKE receiver
🏃 4 — Doppler Shift (Mobility effect)
  • Moving RX → received frequency is shifted: ν = f_c · (v/c) · cos(γ)
  • γ = angle between velocity vector and wave direction
  • Maximum Doppler: ν_max = f_c · v / c
  • Example: GSM 900 MHz, v = 110 km/h → ν_max = 92 Hz
  • Example: WLAN 5.2 GHz, v = 5 km/h → ν_max = 24 Hz
  • Doppler causes time-varying channel: channel "moves" at rate ν_max

Energy Efficiency — Design Imperatives
🔋 Power Sources
  • Rechargeable: smartphones, optimized power management required
  • One-way batteries: IoT sensors, must last years → extreme efficiency
  • Energy harvesting: RF, solar, thermal → batteryless operation
  • Power mains: BSs — can use high power for downlink
⚙️ UE Energy Design Techniques
  • High-efficiency PA: power amplifier is largest consumer
  • CMOS low-power DSP: reduce digital processing energy
  • Adaptive TX power control: only transmit needed power
  • DTX (Discontinuous TX): switch off during silence
  • Sleep/standby modes: deep sleep between packets
  • High RX sensitivity: GSM operates at −100 dBm
⚠️ The Key Trade-off
  • High sensitivity ↔ energy consumption conflict
  • Low-noise amplifier (LNA) needs bias current → power consumption
  • BS can use more power (mains) → higher downlink power & gain
  • UE safety limits TX power to a few hundred mW (SAR regulation)
  • This asymmetry between UL and DL drives all link budget differences
💡 Expert Summary of Module 2

Every wireless impairment has a specific physical cause and a corresponding mitigation. Fast fading (multipath, scale: λ/2 ~ cm) is mitigated by diversity, OFDM, MIMO. Shadowing (scale: 10–100 m) is handled by power control and fading margin in the link budget. ISI (wideband) is solved by OFDM cyclic prefix or equalizers. Doppler (mobility) is tracked by pilot-aided channel estimation. Understanding this cause-mitigation map is essential for system design.

The Central Question: What is the SNR at the Receiver?
📖 The SNR Chain

The received Signal-to-Noise Ratio (SNR) determines all communication performance (bit error rate, capacity, coverage). To compute it, we must account for every gain and loss between TX and RX. The link budget is simply the table that lists all of these.

Master SNR Equation SNR [dB] = P_TX [dBm] + G_TX [dBi] − L_feeder_TX [dB] − L_path [dB] + G_RX [dBi] − L_feeder_RX [dB] − P_noise [dBm]
where P_noise = −174 + 10·log₁₀(B[Hz]) + NF [dBm]
Noise — From Physics to dBm
🌡️ Thermal Noise — The Fundamental Floor
  • Origin: random thermal motion of electrons in any resistor
  • Noise PSD: N₀ = k_B · T₀ where k_B = 1.38×10⁻²³ J/K
  • At room temperature T₀ = 290 K: N₀ = −174 dBm/Hz
  • Total noise in bandwidth B: P_n = −174 + 10·log₁₀(B) dBm
  • Example: B = 200 kHz (GSM): P_n = −174 + 53 = −121 dBm
  • Example: B = 20 MHz (LTE): P_n = −174 + 73 = −101 dBm
📐 Noise Factor / Figure — Receiver Degradation
  • Noise factor F: ratio of output SNR to input SNR (linear, ≥1)
  • Noise figure NF = 10·log₁₀(F) in dB (≥0 dB)
  • Effective noise: P_eff = −174 + 10·log₁₀(B) + NF [dBm]
  • NF of LNA: 1–3 dB (good) · consumer devices: 5–10 dB
  • Man-made noise: +20 dB extra at 150 MHz, +10 dB at 900 MHz, +5 dB (urban UMTS)
  • NF affects only thermal noise — not external interference
Antennas — Directing Energy
⭕ Isotropic Antenna (Reference)
  • Radiates equally in all directions (spherical pattern)
  • Theoretical only — cannot be physically built
  • Gain = 0 dBi by definition
  • Used as reference for all antenna gain specifications
📡 Dipole Antenna
  • λ/2 dipole: gain ≈ 2.15 dBi above isotropic
  • Radiates in a torus shape — no radiation along antenna axis
  • The principle: same input power, more energy in useful directions → apparent "gain"
  • Antenna gain = power focusing, NOT power amplification
⚡ EIRP — Effective Radiated Power
  • EIRP = P_TX + G_TX [dBm + dBi = dBm]
  • Answers: "How much isotropic power gives same max radiation?"
  • Regulatory limit is often on EIRP (not just power)
  • UE real gain is often negative (near-field absorption by body)

Path Loss Models — How Signal Decays with Distance
🌌 Friis' Free-Space Law (d < d_break)
  • P_RX = P_TX · G_TX · G_RX · (λ/4πd)²
  • In dB: P_RX = P_TX + G_TX + G_RX − 20·log₁₀(4πd/λ)
  • Path loss at 1 m, 1 GHz: L_FS(1m, 1GHz) = 32 dB
  • For arbitrary f_c: add 20·log₁₀(f_c / 1 GHz) dB
  • For d > 1 m: add 20·log₁₀(d / 1 m) dB
  • Valid only in the far field: d > d_Rayleigh = 2L_a²/λ
📉 Breakpoint Model (d > d_break)
  • In real terrestrial systems, at d > d_break the path loss exponent changes
  • PL(d) = PL(d_break) + 10·α_LBP · log₁₀(d/d_break)
  • α_LBP: path loss exponent = 1.5–2.5 (LOS) or 3.5–4.5 (NLOS)
  • Physical cause: two-ray ground reflection cancels out (d⁻⁴ law takes over)
  • d_break ≈ 4·h_TX·h_RX / λ (antenna heights)
Complete Path Loss Formula (mental arithmetic version) PL(d, f_c) = 32 + 20·log₁₀(f_c/1GHz) + 20·log₁₀(d/1m) [d ≤ d_break, d in meters] PL(d, f_c) = 32 + 20·log₁₀(f_c/1GHz) + 20·log₁₀(d_break) + α·10·log₁₀(d/d_break) [d > d_break]
Fading Margin — Accounting for Channel Variability
⚠️ Why We Need a Fading Margin

Path loss models give only the mean received power. Due to fading, the actual instantaneous power fluctuates randomly. If we design for the mean, transmission quality is met only 50% of the time — unacceptable. A fading margin (FM) is added to ensure quality in e.g. 90% or 99% of locations/times. Its value depends on the fading statistics (Rayleigh, log-normal shadowing, etc.).

💡 Expert Summary of Module 3

The link budget is the central engineering tool. Key rule: all quantities in dB, then just add/subtract. The noise floor is always −174 + 10·log₁₀(B) + NF dBm. Path loss at 1 m, 1 GHz = 32 dB (memorize this). EIRP = P_TX + G_TX captures effective radiated power. The fading margin converts a "50% quality" design into a "90% or 99% quality" design. UL is often the limiting link due to lower UE TX power and higher UE noise figure.

Six Propagation Mechanisms — The Complete Picture
☀️ 1. Free-Space Propagation
  • No obstacles → spherical wave spreading
  • Power falls as d⁻² (inverse square law)
  • Friis: P_RX = P_TX·G_TX·G_RX·(λ/4πd)²
  • Valid only in far field: d > d_R = 2L_a²/λ
  • Reference for all other losses
🪞 2. Reflection & Transmission
  • Wave hits boundary (wall, ground): part reflected, part transmitted
  • Reflection angle = incidence angle (Snell's law)
  • Separate treatment for TE (E⊥ to plane) and TM (E∥ to plane)
  • Brewster angle: TM reflection = 0 (total transmission)
  • At grazing incidence: TE reflection ≈ 1, phase inverted
🏢 3. Transmission Through Walls
  • Wall = layered dielectric → cascaded reflection+transmission
  • Concrete/steel at 2 GHz: ~20 dB loss
  • Brick/stone at 2 GHz: ~10 dB loss
  • Drywall at 2 GHz: ~3 dB loss
  • At mm-wave (5G): concrete/steel up to 60 dB — essentially impenetrable!
  • Energy-saving windows (metal coating): +40–60 dB at mm-wave
↩️ 4. Diffraction (Going Behind Corners)
  • Waves bend around edges of finite objects (buildings, hills)
  • Based on Huygens' principle: every wavefront point = secondary source
  • Explained by Fresnel integral with parameter ν_F
  • Less pronounced as λ decreases (more ray-like at high freq)
  • Critical for coverage behind buildings — the "why urban coverage works"
🌀 5. Scattering (Rough Surfaces)
  • Rough surfaces scatter energy in multiple directions
  • Smooth surface: specular reflection (like mirror)
  • Rough surface: diffuse scattering (Lambertian, single-lobe, double-lobe models)
  • Kirchhoff theory: effective ρ_rough = ρ_smooth · exp(−2(k₀σ_h sinψ)²)
  • σ_h = surface height deviation, k₀ = 2π/λ
🌧️ 6. Atmospheric Attenuation
  • Significant only at mm-wave and above
  • O₂ absorption peak at 60 GHz (~10–15 dB/km)
  • H₂O absorption peaks at 22 GHz and 183 GHz
  • 5G 57–71 GHz band: high atmospheric loss → short-range only
  • Can be exploited for frequency reuse (natural isolation)
The d⁻⁴ Two-Ray Power Law — When Ground Matters
📉 Two-Ray Model (LOS + Ground Reflection)
  • Scenario: direct LOS path + single ground-reflected path
  • For d ≪ d_break: oscillating pattern (constructive/destructive alternation)
  • For d ≫ d_break: smooth d⁻⁴ decay
  • P_RX ∝ (h_TX · h_RX)² / (λ² · d⁴)
  • Breakpoint: d_break = 4·h_TX·h_RX / λ
  • Example: h_TX = 30 m, h_RX = 1.5 m, f = 1 GHz → d_break ≈ 600 m
⚠️ Two Types of Breakpoints (Important Distinction)
  • d_break (phase): where direct and reflected waves phase difference = π → first destructive interference
  • d'_break (intersection): where d⁻² and d⁻⁴ curves intersect numerically
  • These two definitions give similar values but have different physical meanings
  • In practice: α_LBP = 1.5 to 5.5 — not always exactly 4
  • A second breakpoint may exist (beyond radio horizon, α > 6)
Diffraction — Knife-Edge and Fresnel Zones
🔪 Knife-Edge Diffraction (Single Screen)
  • Total field: E = e^(−jk₀x) · F̃(ν_F) via Fresnel integral
  • Fresnel parameter: ν_F = −Y·√(2/λx) or for general geometry:
  • ν_F = θ_d · √(2·d_TX·d_RX / λ(d_TX+d_RX))
  • θ_d = angle between LOS and diffracted ray
  • Diffraction LOSS depends on ν_F: ν_F < 0 → small loss; ν_F > 1 → significant loss
🥚 Fresnel Zones — Quick Assessment
  • i-th Fresnel ellipsoid: all reflections add extra path length of i·λ/2
  • Rule: if object does NOT enter the first Fresnel zone → negligible diffraction loss
  • Object in 1st Fresnel zone → loss increases rapidly
  • Practical use: check if obstacles clear the 1st Fresnel zone for reliable LOS link
  • 1st Fresnel zone radius at midpoint: r₁ = √(λ·d_TX·d_RX/(d_TX+d_RX))
Multiple Screen Diffraction — Urban Scenarios
🏙️ Bullington's Method
  • Replaces all screens with one equivalent screen
  • Derived from steepest tangents from TX and RX to obstacles
  • Simple: single diffraction calculation
  • Inaccurate: can be overly optimistic (ignores intermediate obstacles)
  • Total loss: independent of number of screens → wrong functional form
📐 Epstein–Petersen Method
  • Each screen computed individually with virtual TX/RX at ends
  • Losses summed logarithmically: L_tot = L₁ + L₂ + L₃ [dB]
  • Better than Bullington: exponential increase in loss with N screens
  • Limitation: assumes RX is in far field of each screen
  • More accurate for small number of screens
🔬 Method Comparison
  • Slope diffraction (≤15 screens): linear increase in loss — most accurate
  • Modified ITU method: also linear — correct trend
  • Deygout / E-P methods: exponential increase — overestimates for many screens
  • Bullington: independent of N — wrong completely for many screens
  • For 1–3 screens in practice: E-P and Deygout are acceptable
Deterministic Channel Modeling — Ray Tracing & Launching
🚀 Ray Launching (Shoot & Bounce)
  • Launch rays in all directions from TX, trace their paths
  • Follow each ray through reflections, transmissions, diffractions
  • Compute amplitude, delay, DoD, DoA for all rays reaching RX
  • Advantage: one simulation → all RX positions in coverage map
  • Ray splitting: keep angular resolution constant — prevents energy spreading
  • Accuracy: 3–6 dB mean error (received power) at 0.5–2.5 GHz
📍 Ray Tracing (Image Principle)
  • Find all geometric paths between one TX-RX pair
  • Single reflection: create virtual source (image of TX in reflector)
  • Double reflection: image of image, etc.
  • More accurate but: one simulation per TX-RX pair
  • Each MPC: amplitude (GTD/UTD coefficients), delay (path length), DoD, DoA, polarization
  • Uses geographic database: indoor (floor plans), urban (LIDAR point clouds)
💡 Expert Summary of Module 4

The radio channel is shaped by six mechanisms. The most important are: (1) free-space spreading (d⁻², Friis), (2) ground reflection leading to d⁻⁴ decay beyond the breakpoint, and (3) diffraction enabling coverage behind buildings. Penetration loss at mm-wave (5G) is so severe that new cell architecture (dense small cells, beamforming) is required. Scattering contributes to angular spread (critical for MIMO). Deterministic models (ray tracing) achieve 3–6 dB accuracy but need geographic databases. All MPCs they produce form the basis of the statistical models in Chapter 5.

The Hierarchical Fading Model
📖 Three Levels of Signal Variation
  • Path Loss: deterministic decay with distance (d⁻² to d⁻⁴) — scale: km
  • Shadowing (Large-scale fading): random log-normal variation from obstacles — scale: 10–100 m
  • Small-scale (Fast) fading: rapid variation from multipath interference — scale: λ/2 ≈ cm

The received power at any point = mean path loss + shadowing deviation + fast fading fluctuation. All three are independent and superimposed.

Derivation of Rayleigh Fading — The CLT Argument
🔄 Step 1 — Many MPCs as Complex Phasors
  • N incoming waves (MPCs) with amplitudes a_ℓ and random phases φ_ℓ
  • Total received field: E = Σ a_ℓ · e^(jφ_ℓ)
  • Re(E) = Σ a_ℓ cos(φ_ℓ), Im(E) = Σ a_ℓ sin(φ_ℓ)
  • Each term is a random variable, statistically independent
📊 Step 2 — CLT → Gaussian → Rayleigh
  • No dominant MPC → phases φ_ℓ uniformly distributed in [0,2π]
  • By Central Limit Theorem: Re(E) and Im(E) → independent Gaussian N(0,σ²)
  • Amplitude r = |E| = √(Re² + Im²) follows Rayleigh distribution
  • Phase arg(E) → uniformly distributed on [0,2π]
  • This is the fundamental mathematical justification — not just an empirical fit
Rayleigh Distribution — NLOS Scenario
📐 Rayleigh PDF & CDF
  • PDF: p(r) = (r/σ²)·exp(−r²/2σ²) for r ≥ 0
  • CDF: P(r < r_min) = 1 − exp(−r_min²/2σ²)
  • Mean: E[r] = σ√(π/2) ≈ 1.25σ
  • RMS value: r_rms = σ√2 → P̄ = E[r²] = 2σ²
  • Variance: Var(r) = (2−π/2)σ² ≈ 0.429σ²
  • Median: r_50 = σ√(2·ln2) ≈ 1.18σ
🛡️ Fading Margin — Quantitative Derivation
  • For outage probability x (probability signal is below threshold):
  • P(P < P_min) = 1 − exp(−P_min/P̄) ≈ P_min/P̄ for small x
  • Required mean power: P̄ = P_min / x
  • Fading margin: FM = −10·log₁₀(x) [dB]
  • 1% outage (x = 0.01): FM = 20 dB
  • 10% outage (x = 0.1): FM = 10 dB
  • 0.1% outage (x = 0.001): FM = 30 dB

Rice Distribution — LOS Scenario
☀️ When a Dominant Component Exists
  • One component has amplitude A (LOS or strong specular reflection)
  • Re(E) ~ N(A, σ²) — non-zero mean; Im(E) ~ N(0, σ²)
  • Amplitude follows Rice (Rician) distribution
  • K-factor: K = A²/(2σ²) = LOS power / diffuse power
  • K = 0 → reduces to Rayleigh (NLOS)
  • K → ∞ → approaches AWGN (no fading, all LOS)
  • Typical K: 6–20 dB for indoor LOS environments
🔬 Nakagami Distribution — General Case
  • More general than Rayleigh/Rice — useful when neither fits exactly
  • Parameter m controls fading severity
  • m = 1 → Rayleigh; m = 1/2 → one-sided Gaussian; m → ∞ → AWGN
  • Power P = r² follows a Gamma distribution
  • Approximation: m ≈ (K+1)²/(2K+1) relates to Rice K-factor
  • Used when the environment cannot be cleanly classified as LOS or NLOS
Doppler Effects — The Channel in Time
🏃 Doppler Shift — Single Path
  • RX moving at speed v at angle γ to wave direction:
  • ν = −f_c · (v/c) · cos(γ)
  • Received frequency: f = f_c + ν
  • Max Doppler: ν_max = f_c · v / c
  • Numerical examples:
  • 5.2 GHz WLAN, v=5 km/h (1.4 m/s): ν_max = 24 Hz
  • GSM 900 MHz, v=110 km/h (30.6 m/s): ν_max = 92 Hz
📊 Jakes Doppler Spectrum (Isotropic Scattering)
  • Waves arrive from all directions uniformly → specific Doppler spectrum shape
  • S_D(ν) = 1.5·Ω̄ / (π·√(ν_max² − ν²)) for |ν| ≤ ν_max
  • U-shaped: most energy near ±ν_max (horizontal wave arrivals have max Doppler)
  • Singularity at ±ν_max (integrable — finite power)
  • Time correlation of channel ↔ Fourier transform of Doppler spectrum
  • Decorrelation distance: ~0.18λ (50% correlation drop) to ~0.38λ (full decorrelation)
Level Crossing Rate & Fading Dip Statistics
📉 Level Crossing Rate (LCR)
  • LCR: rate at which amplitude crosses a threshold r_th in the upward direction
  • For Rayleigh + Jakes: LCR = √(2π)·ν_max·(r_th/r_rms)·exp(−r_th²/r_rms²)
  • Deep fades: less frequent but predictable from this formula
  • Fast-moving UE: more crossings per second, but shorter dip duration
  • Application: error burst analysis, interleaver design
🏢 Large-Scale Fading — Lognormal Shadowing
  • Small-scale averaged power follows: S_dB ~ N(0, σ_S²)
  • Field strength in linear scale → lognormal distribution
  • Shadowing is correlated in space: Gudmundson model
  • R_shad(Δ) = exp(−Δ/d_shad-cor)
  • Correlation distance: d_shad-cor ≈ 10 m (urban) to 100 m (rural)
  • σ_S: 1–4 dB (LOS, indoor) to 6–12 dB (dense urban NLOS)
💡 Expert Summary of Module 5

Statistical modeling replaces deterministic ray tracing with probability distributions. The key insight: many independent MPCs (CLT) → Gaussian complex amplitude → Rayleigh envelope. This is a derivation, not a postulate. The fading margin formula FM = −10·log₁₀(outage probability) makes the link budget probabilistic. Rice fading (LOS) is less severe than Rayleigh (K > 0 reduces the fading depth). The Doppler spectrum (Jakes U-shape) characterizes how fast the channel changes in time. Large-scale shadowing adds a separate lognormal layer that must also be margined.

From Narrowband to Wideband — Why We Need a New Framework
📖 The Narrowband Assumption Breaks Down

In Module 5, we assumed the channel is a single complex gain: y(t) = A(t)·e^(jφ(t))·x(t). This is valid only when all MPCs arrive within one symbol period — the narrowband (flat-fading) assumption. When the signal bandwidth is large enough to resolve individual MPCs (wideband), each frequency component sees a different channel gain. We need a much richer mathematical framework.

The Two-Path Model — Gateway to Wideband
⚡ Channel Impulse Response
  • h(τ) = a₁δ(τ−τ₁) + a₂δ(τ−τ₂)
  • aₖ = |aₖ|e^(jφₖ): complex amplitude of path k
  • τₖ = dₖ/c₀: delay from path length dₖ
  • Static scenario → LTI channel
  • Transfer function H(f) = FT of h(τ)
📊 Transfer Function — Frequency-Selective Fading
  • H(f) = a₁e^(−j2πfτ₁) + a₂e^(−j2πfτ₂)
  • |H(f)|² = |a₁|² + |a₂|² + 2|a₁||a₂|cos(2πfΔτ − Δφ)
  • Oscillates in frequency with period Δf = 1/Δτ (notch spacing)
  • Δτ = τ₂ − τ₁: differential delay between paths
  • Example: Δτ = 1 μs → notch every 1 MHz
  • Group delay: τ_Gr = −(1/2π)·d(arg H)/df → large at fading notches → ISI
The Four Equivalent System Descriptions
🔄 The Bello System Functions (Fundamental Framework)

For a time-variant channel h(t,τ), Fourier transforms can be applied with respect to t (→ Doppler domain ν) and τ (→ frequency domain f), giving four equivalent descriptions. Each provides different physical insight.

The WSSUS Model — Simplifying the Stochastic Channel
🎲 From Deterministic to Stochastic
  • Real channels are stochastic: h(t,τ) is a 2D random process
  • Full description: ACF R_h(t,t';τ,τ') depends on 4 variables — intractable
  • WSSUS = two simplifying assumptions that reduce 4 → 2 variables
  • WSS (Wide-Sense Stationary): ACF depends only on time difference Δt = t'−t
  • US (Uncorrelated Scattering): different delay taps are uncorrelated
  • Together: R_h(t,t';τ,τ') = P_h(Δt,τ)·δ(τ−τ')
🏗️ Tapped Delay Line (TDL) Model
  • WSSUS channel → FIR filter with time-varying taps:
  • h(t,τ) = Σ cᵢ(t)·δ(τ−τᵢ)
  • N taps at fixed delays τᵢ (spaced 1/B apart)
  • Each tap cᵢ(t): complex Gaussian process with Doppler spectrum
  • Power of tap i: σᵢ² = P_h(τᵢ) from Power Delay Profile
  • This is the basis for all standardized channel simulation models

Double-Directional Channel — For MIMO Systems
🎯 Beyond SISO: Directional Information
  • SISO channel h(t,τ): no directional info (antenna effects included)
  • For MIMO: need to separate channel from antenna patterns
  • Double-directional impulse response: h(t,τ,Ω,Ψ)
  • Ω = Direction of Departure (DoD) from TX
  • Ψ = Direction of Arrival (DoA) at RX
  • Each MPC described by: (aₗ, τₗ, Ωₗ, Ψₗ, νₗ, polarization)
📐 Angular Spread & MIMO Performance
  • Angular spread (Fleury def.): RMS of angular power spectrum — avoids ambiguity of circular angle
  • Large angular spread → MPCs arrive from many directions → high MIMO multiplexing gain
  • Small angular spread (LOS) → low MIMO rank → beamforming better than multiplexing
  • MIMO channel matrix H = integral of h(τ,Ω,Ψ)·steering vectors over Ω,Ψ
  • Spatial multiplexing gain limited by min(N_TX, N_RX, #independent paths)
💡 Expert Summary of Module 6

Wideband channels require moving from a scalar gain to a full impulse response h(t,τ). The four Bello system functions (h,H,s,B) are all mathematically equivalent — choose based on the problem. WSSUS reduces the intractable 4D ACF to 2D via two physical assumptions (stationarity + uncorrelated delays). The five condensed parameters (PDP, S_τ, B_c, B_D, T_c) are the engineering-relevant summary of any channel. The critical design rule: OFDM cyclic prefix must exceed S_τ; pilot spacing must track B_D. For MIMO: angular spread determines spatial multiplexing gain.

Two Paradigms of Channel Modeling
🗺️ Location-Specific Models (Deterministic)
  • Purpose: network planning, site-specific deployment
  • Based on: geographic database + ray tracing/launching
  • Accuracy: 3–6 dB mean error in received power
  • Inputs: 3D building map, material properties, terrain data
  • Output: coverage map, delay spread map, interference estimate
  • Cost: high computation; requires LIDAR or GIS database
  • Not suitable for: algorithm design or standardized testing
🎲 Stochastic Models (Statistical)
  • Purpose: algorithm design, testing, type approval
  • Based on: measurement campaigns → statistical fitting
  • Location-independent: parameterized for "urban", "indoor", etc.
  • Standardized by: 3GPP, ITU, IEEE — reproducible across all vendors
  • Enables: fair comparison between different implementations
  • Models: TDL (Tapped Delay Line), CDL (Clustered Delay Line)
  • Limitation: may not represent specific deployment accurately

Delay Dispersion Models
📊 Exponential PDP — Standard Model
  • Single-cluster: P(τ) = e^(−τ/S_τ) for τ ≥ 0
  • Captures: one-sided exponential decay of received power vs delay
  • Multi-cluster: sum of delayed exponentials
  • Physical basis: more obstacles with longer paths → more attenuation
  • RMS delay spread S_τ is the single free parameter
🔬 Saleh-Valenzuela (SV) Model
  • MPCs arrive in clusters (not independently)
  • Cluster arrival: Poisson with rate Λ (inter-cluster gaps)
  • Ray arrival within cluster: Poisson with rate λ (intra-cluster gaps)
  • Power: decays exponentially within cluster and cluster-to-cluster
  • Physical cause: clusters = groups of reflectors at similar geometry
  • Standard for UWB (IEEE 802.15.4a) and mmWave 5G channels

Semi-Deterministic Models — Best of Both Worlds
🔬 Hybrid Approach (COST 2100, 3GPP Geometry-Based)
  • D-rays (Deterministic): computed via simplified ray tracing from large fixed structures (walls, buildings) — strong, specular reflections
  • Cluster MPCs: around each D-ray, random spread in delay and angle — models diffuse scattering
  • R-rays (Random): weak random multipaths not derived from geometry — environmental randomness

  • Used in: 3GPP TR 38.901 (5G channel model), COST 2100, WINNER II/+
💡 Expert Summary of Module 7

The right model depends on the use case: ray tracing for site-specific planning, stochastic TDL/CDL models for standardized algorithm testing. The pathloss model captures the mean behavior; shadowing (lognormal, σ_S = 4–12 dB) adds slow random variation; the TDL captures delay dispersion. RMS delay spread (5 ns indoor to 18 μs hilly terrain) directly determines whether ISI occurs and how large the OFDM guard interval must be. The 3GPP 5G channel models (TDL-A/B/C, CDL-A/B/C) extend this to mmWave and directional channels for Massive MIMO design.

OFDM — The Channel Taming Technology
Used in: LTE (4G), 5G NR, Wi-Fi (802.11a/g/n/ac/ax), DVB-T/T2, ADSL. The dominant wideband modulation standard since 2000s. Directly solves ISI from delay spread.
🎯 Core Problem OFDM Solves
  • High data rate → short symbol duration T_s → ISI when T_s < S_τ
  • Equalizer complexity grows as channel gets more frequency-selective
  • OFDM idea: instead of one fast symbol, use N slow parallel symbols
  • Each subcarrier has long symbol duration: T_OFDM = N·T_s + T_CP
  • If T_CP > τ_max → no ISI between adjacent OFDM symbols
  • Each subcarrier sees a flat-fading channel (simple 1-tap equalizer)
🔢 OFDM Numerical Example
  • Problem: 10 Msymbol/s in urban channel with τ_rms = 250 μs
  • Single carrier: T_s = 100 ns → K = τ_rms/T_s = 2500 interfering symbols!
  • OFDM with N = 2048 subcarriers: T_sub = 2048 × 100 ns = 204.8 μs
  • Add CP: T_CP = 250 μs > τ_max → ISI eliminated completely
  • Each subchannel bandwidth: 1/T_sub ≈ 5 kHz ≪ B_c → flat fading
  • Overhead: T_CP/(T_sub + T_CP) ≈ 55% → significant! Choose N wisely
⚙️ OFDM Modulation — Mathematical Core
  • OFDM signal: x(t) = Σ Xₖ·exp(j2πk·t/T_OFDM), k = 0,...,N−1
  • Subcarriers are mutually orthogonal: ∫ e^(j2πkt/T)·e^(−j2πmt/T) dt = T·δ(k−m)
  • Transmitter: IDFT of frequency-domain symbols → time-domain waveform
  • Receiver: DFT of received waveform → frequency-domain symbols
  • Add CP (copy of last part of IFFT output → prepend to block)
  • Implemented efficiently via IFFT/FFT (N log₂N operations vs N² for DFT)
✅ OFDM Pros and Cons
  • ✅ Eliminates ISI with CP — no equalizer needed per subcarrier
  • ✅ Simple frequency-domain equalization: divide by H(f_k) per subcarrier
  • ✅ Flexible resource allocation: assign subcarriers per user (OFDMA)
  • ✅ Compatible with MIMO and beamforming
  • ❌ High Peak-to-Average Power Ratio (PAPR): requires linear PA
  • ❌ Sensitive to frequency offset (breaks orthogonality) — needs synchronization
  • ❌ CP overhead reduces spectral efficiency
Spread Spectrum & CDMA — Sharing the Spectrum with Codes
Spread spectrum intentionally uses more bandwidth than needed. Used in: 3G UMTS/CDMA2000, GPS, military communications, Wi-Fi (2.4 GHz), UWB (IEEE 802.15.4a). Key advantages: interference resistance, multi-user sharing, multipath diversity.
📡 Frequency Hopping (FH-SS)
  • Carrier frequency changes rapidly per pseudorandom code
  • Signal occupies vulnerable frequency only briefly
  • Slow hopping: same freq for ≥1 symbol
  • Fast hopping: multiple hops per symbol
  • Anti-jamming: jammer cannot track hopping pattern
  • Collision design: Hamming distance of hopping sequences minimizes overlap
💻 DS-CDMA (Direct Sequence)
  • Info bit (rate R_b) × PN code (rate R_c = M_c·R_b)
  • Spreading factor M_c = R_c/R_b = T_b/T_c
  • Transmitted BW ≈ 1/T_c = M_c/T_b ≫ original BW
  • Receiver: despread by multiplying same code → narrows back to signal BW
  • Narrowband interference gets spread → interference becomes noise-like
  • Multiple users: different PN codes → simultaneous sharing
📶 Rake Receiver — Exploiting Multipath
  • PN code with high M_c can resolve individual multipath components!
  • Multiple "fingers" — each correlator tuned to one MPC delay
  • Combine all fingers: Maximum-Ratio Combining (MRC)
  • Turns multipath (impairment!) into diversity gain
  • All-Rake (ideal): combine all paths; Selective/Partial Rake: practical
🆘 Near-Far Problem — Critical CDMA Issue
  • Nearby user's signal overwhelms far user at the BS
  • All users transmit on same frequency → mutual interference
  • Solution: closed-loop uplink power control
  • BS measures received power, sends power-down commands every ms
  • Target: all users arrive at BS with equal power (within ±1 dB)
  • Accuracy requirement: very tight — CDMA capacity collapses without it
📊 CDMA Soft Capacity
  • No hard user limit — interference increases gracefully ("noise rise")
  • System "full" when noise rise ≈ 6 dB → ~75% of pole capacity
  • Code families: m-sequences (ACF ≈ delta), Gold (many codes, good CCF), Kasami, Walsh-Hadamard (perfectly orthogonal DL)
  • DL: Walsh-Hadamard codes → orthogonal → no intra-cell interference
  • UL: approximately orthogonal → CDMA uplink capacity lower than DL
Ultra-Wideband (UWB) — Pulses for Localization
⚡ UWB Definition & Properties
  • Bandwidth: >500 MHz absolute or >20% relative bandwidth
  • Very short pulses → cm-level time-of-flight ranging
  • Low power spectral density (below noise floor of other systems)
  • High peak-to-average ratio → multiple pulses per symbol needed
  • Applications: indoor localization (phones, cars, Apple AirTag), radar, imaging
🏓 Time-Hopping Impulse Radio (TH-IR)
  • Symbol time split into frames T_f; pulse position within frame shifted by code c_j·T_c
  • Multiple pulses per symbol → solves peak-power problem
  • Users assigned different hopping codes → no "catastrophic collision"
  • Coherent detection: Rake receiver (20+ fingers indoors)
  • Transmitted Reference (TR): reference pulse + data pulse — simple but 3 dB energy loss
  • Non-coherent energy detection: simplest — but strong MAI sensitivity

💡 Expert Summary of Module 8

OFDM is the dominant solution to ISI: it converts the frequency-selective wideband channel into N flat-fading subchannels, enabling simple per-subcarrier equalization. The CP length must exceed the maximum excess delay. DS-CDMA trades bandwidth for interference resistance and turns multipath into diversity via the Rake receiver — but requires strict power control to solve the near-far problem. UWB uses sub-nanosecond pulses for cm-scale ranging; TH-IR provides multiple-access UWB with good anti-collision properties. The progression CDMA → OFDMA reflects the trade-off: CDMA excels in interference environments, OFDMA excels in frequency-selective wideband channels with flexible user scheduling.
